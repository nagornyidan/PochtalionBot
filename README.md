# 📧 Почтальон Bot

Асинхронный Telegram-бот на **Aiogram 3**, который превращает клиент Telegram в почтовый клиент. Бот поддерживает работу с несколькими ящиками одновременно и присылает уведомления о новых письмах в режиме реального времени каждые 5 минут.

## ✨ Основные возможности
* **Мультиаккаунтность:** Добавляйте неограниченное количество ящиков (Gmail, Yandex, Outlook, Mail.ru).
* **Фоновые уведомления:** Автоматическая проверка почты каждые 5 минут с помощью `APScheduler`.
* **Безопасность:** Работа через протокол IMAP SSL и использование "Паролей приложений".
* **Интерфейс:** Удобное меню с использованием Inline-кнопок для чтения заголовков и содержания писем.

## 🛠 Технологический стек
* **Python 3.10+**
* **Aiogram 3.x** (фреймворк для бота)
* **aioimaplib** (асинхронная работа с почтой)
* **SQLite3** (хранение данных аккаунтов)
* **APScheduler** (планировщик задач)

## ⚙️ Настройка почтовых сервисов
Для корректной работы необходимо включить IMAP в настройках вашей почты и создать **Пароль приложения**.

| Сервис | Что нужно сделать |
| :--- | :--- |
| **Gmail** | Включить 2FA -> Создать "App Password" -> Включить IMAP в настройках Gmail. |
| **Yandex** | Настройки -> Почта -> Программы -> Разрешить доступ по IMAP -> Создать пароль приложения. |
| **Outlook** | Параметры -> Почта -> Пересылка и POP/IMAP -> Разрешить IMAP. Использовать 2FA + App Password. |
| **Mail.ru** | Настройки -> Пароли и безопасность -> Пароли для внешних приложений. |

## 📂 Структура базы данных
Бот использует SQLite (`users.db`). При первом запуске создается таблица `accounts_v5` со следующими полями:
* `user_id`: Telegram ID пользователя.
* `email`: Адрес электронной почты.
* `password`: Пароль приложения (зашифрован протоколом SSL при передаче).
* `service`: Тип сервиса (для выбора хоста).
* `last_uid`: ID последнего полученного письма (для системы уведомлений).

## ⚠️ Известные проблемы и решения

### Microsoft Outlook (Basic Auth Blocked)

С конца 2024 года Microsoft начала массово отключать поддержку **Basic Authentication** (простая связка логин + пароль) для персональных аккаунтов. Это самая частая причина ошибок при работе бота.

**Симптомы:**

  * Бот выдает ошибку аутентификации.
  * В консоли логов (Debug) виден код ошибки `NO` или `NONAUTH`.

**Как попробовать исправить:**

1.  **Двухфакторная аутентификация (2FA):** Она должна быть **обязательно включена** в вашем аккаунте Microsoft.
2.  **Пароль приложения:** Обычный пароль от почты не сработает. Создайте свежий 16-значный «Пароль приложения» в разделе безопасности.
3.  **Журнал действий:** Зайдите в [Account Security Activity](https://www.google.com/search?q=https://account.microsoft.com/activity) и, если видите заблокированную попытку входа от бота, нажмите **«Это был я»**.
4.  **Региональные ограничения:** Microsoft может блокировать подключения, если сервер бота находится в другой стране. Попробуйте запустить бота локально для проверки.

> **Примечание:** Если после всех манипуляций Outlook продолжает выдавать `BasicAuthBlocked`, это означает, что ваш аккаунт полностью переведен на **OAuth2**. В текущей версии бот работает через IMAP-авторизацию, которая может быть недоступна для таких аккаунтов.

### Общие проблемы

  * **Gmail:** Если бот не видит письма, убедитесь, что в настройках Gmail (веб-версия) во вкладке «Пересылка и POP/IMAP» выбран пункт **«Включить IMAP»**.
  * **База данных:** При обновлении бота с добавлением новых функций (например, уведомлений) рекомендуется удалить старый файл `users.db`, чтобы таблица пересоздалась с нужными колонками.
  * **Задержки:** Если у вас добавлено более 10-20 ящиков, интервал проверки в 5 минут может вызывать повышенную нагрузку. Рекомендуется увеличить интервал в `scheduler.add_job`.
